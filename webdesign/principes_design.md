# Principes fondamentaux du design en webdesign

## Source

- https://www.youtube.com/watch?v=A8bsrYqn0NQ

## Introduction

Les bases du design reposent sur quelques principes universels utilisés dans presque toutes les disciplines visuelles : graphisme, UI design, webdesign et édition.

Ces principes permettent de créer un site :

- lisible
- équilibré
- intuitif
- esthétique
- cohérent

Les cinq principes fondamentaux sont :

1. l'équilibre
2. l'alignement
3. la proximité
4. la répétition
5. le contraste

Certains designers utilisent l'acronyme **CRAP** pour parler du contraste, de la répétition, de l'alignement et de la proximité. Cet acronyme est utile, mais il oublie l'équilibre, qui reste essentiel en webdesign.

## 1. L'équilibre

### Définition

L'équilibre consiste à répartir le poids visuel des éléments sur une page pour éviter qu'une zone paraisse trop chargée et une autre trop vide.

Un design équilibré favorise :

- une lecture naturelle
- une navigation fluide
- une impression d'harmonie

À l'inverse, un mauvais équilibre crée une sensation d'inconfort ou de confusion.

### Les principaux types d'équilibre

#### Équilibre symétrique

Les éléments sont disposés de manière miroir.

Exemple :

```text
image | texte
image | texte
```

On le retrouve souvent sur :

- les sites institutionnels
- les landing pages classiques

Avantages :

- stable
- rassurant

Limite :

- peut paraître rigide

#### Équilibre asymétrique

Les éléments sont différents, mais leur poids visuel reste compensé.

Exemple :

```text
image large | texte + bouton
```

C'est un équilibre très utilisé dans les interfaces modernes.

Avantages :

- dynamique
- plus vivant visuellement

#### Équilibre radial

Les éléments sont organisés autour d'un centre.

Exemples :

- roue
- diagramme
- menu circulaire

Ce type d'équilibre est plus rare sur le web.

### Exemple concret

Bon équilibre :

```text
[Image large]     [Titre]
                  [Texte]
                  [Bouton]
```

Mauvais équilibre :

```text
[Titre] [Bouton] | [Image énorme + texte]
```

Dans le second cas, le poids visuel est mal réparti.

## 2. L'alignement

### Définition

L'alignement consiste à organiser les éléments selon des lignes invisibles. Chaque bloc doit sembler relié aux autres. Rien ne doit donner l'impression d'être placé au hasard.

### Pourquoi c'est important

Un mauvais alignement donne une impression :

- d'amateurisme
- de désordre
- de manque de structure

Un bon alignement :

- facilite la lecture
- crée un rythme visuel
- renforce la compréhension

### Alignements courants

#### Alignement à gauche

C'est l'alignement le plus lisible pour les textes longs.

```text
Titre
Texte
Texte
Texte
```

#### Alignement centré

Souvent utilisé pour :

- les hero sections
- les citations
- certaines landing pages

Il fonctionne bien sur des contenus courts, mais devient moins lisible sur de longs paragraphes.

#### Alignement à droite

Il est plus rare et sert surtout pour :

- des effets de composition
- des éléments décoratifs

### Règle UX importante

Le contenu principal doit rester simplement aligné.

Les alignements plus créatifs doivent surtout être réservés :

- aux images
- aux éléments décoratifs
- à certains effets visuels

## 3. La proximité

### Définition

La proximité consiste à regrouper les éléments qui ont un lien logique.

Les éléments liés doivent être proches. Les éléments non liés doivent être séparés.

### Exemple simple

Mauvaise proximité :

```text
Titre


Texte
```

Le lien entre le titre et le texte devient moins évident.

Bonne proximité :

```text
Titre
Texte
```

Le cerveau comprend immédiatement qu'ils appartiennent au même bloc.

### Application en UX

La proximité aide à structurer :

- les cartes
- les menus
- les formulaires
- les sections de contenu

Exemple :

```text
Produit
prix
bouton acheter
```

Ces trois éléments doivent être perçus comme un ensemble.

### Lien avec l'espace blanc

La proximité est directement liée à l'espace blanc.

L'espace blanc permet :

- de séparer les sections
- de clarifier la structure
- d'améliorer la lisibilité

## 4. La répétition

### Définition

La répétition consiste à réutiliser les mêmes styles et les mêmes règles visuelles dans tout le site.

Elle crée :

- de la cohérence
- une identité visuelle stable
- une expérience utilisateur plus prévisible

### Éléments que l'on répète souvent

Dans un site web, on répète par exemple :

- la typographie
- les couleurs
- les boutons
- les icônes
- les marges
- les espacements
- les grilles

### Exemple

Tous les titres de même niveau doivent suivre la même logique visuelle.

```text
H1
H1
H1
```

Tous les boutons primaires doivent aussi rester cohérents.

```text
[ Acheter ]
[ Acheter ]
[ Acheter ]
```

### Pourquoi c'est important

Sans répétition :

- le site semble incohérent
- l'utilisateur doit réapprendre l'interface

Avec répétition :

- la navigation devient plus intuitive
- les repères visuels sont plus forts
- l'identité du site est plus claire

## 5. Le contraste

### Définition

Le contraste permet de mettre en évidence les éléments importants. Il sert à créer une hiérarchie visuelle claire.

### Les principales formes de contraste

#### Contraste de taille

```text
H1 énorme
texte normal
```

Il permet de hiérarchiser rapidement les informations.

#### Contraste de couleur

Exemple :

```text
fond bleu
texte blanc
```

Un bouton ou un bloc peut ainsi ressortir visuellement.

#### Contraste de poids

```text
bold
normal
light
```

Le poids typographique aide à distinguer ce qui est principal de ce qui est secondaire.

#### Contraste de forme

Exemple :

```text
bouton rond
carte rectangulaire
```

La différence de forme peut attirer l'attention ou clarifier un usage.

### Contraste et accessibilité

Le contraste est essentiel pour :

- la lisibilité
- l'accessibilité
- le respect des normes WCAG

Exemples fréquents :

```text
texte noir
fond blanc
```

ou

```text
texte blanc
fond sombre
```

## Pourquoi ces principes sont fondamentaux

Ces cinq principes agissent à deux niveaux :

### UI

Ils améliorent l'apparence de l'interface.

### UX

Ils facilitent l'utilisation et la compréhension.

Un bon design n'est donc pas seulement esthétique. Il doit aussi être fonctionnel, intuitif et efficace.

## La hiérarchie visuelle

La hiérarchie visuelle est la manière dont un design guide le regard de l'utilisateur.

Quand une personne arrive sur une page web, elle ne lit pas tout. Elle scanne rapidement l'écran pour identifier :

- les éléments importants
- les actions possibles
- les informations utiles

La question centrale est simple : qu'est-ce que l'utilisateur doit voir en premier, en deuxième, puis en dernier ?

Un design sans hiérarchie visuelle oblige l'utilisateur à chercher l'information, ce qui crée de la friction.

### Les outils de la hiérarchie visuelle

#### La taille

Les éléments les plus grands attirent naturellement le regard.

```text
Titre principal
Sous-titre
Texte
```

Dans une page classique :

- le H1 est très visible
- le H2 est secondaire
- le texte apporte le détail

#### Le contraste

Un élément qui contraste avec son environnement attire immédiatement l'oeil.

Exemple :

- page bleue
- bouton orange

Le bouton devient alors un point d'attention naturel. C'est souvent le rôle du CTA.

#### La position

Les éléments placés :

- en haut
- au centre
- ou isolés

sont généralement vus en premier.

#### L'espace autour d'un élément

Un élément entouré d'espace devient plus visible.

```text
texte texte texte texte texte texte

        BOUTON

texte texte texte texte texte texte
```

L'espace libre aide à en faire un point focal.

## Les parcours de lecture : F-pattern et Z-pattern

Les utilisateurs ne lisent pas une page web comme un livre. Ils suivent souvent des schémas de lecture rapides.

### Le F-pattern

Ce comportement est fréquent sur :

- les articles
- les blogs
- les pages riches en texte

Le regard suit souvent cette logique :

```text
██████████████

██████

████
```

En pratique :

1. lecture horizontale en haut
2. seconde lecture horizontale plus courte
3. balayage vertical sur la gauche

Conséquences pour le design :

- placer les informations importantes en haut
- soigner la colonne de gauche
- privilégier les titres, les listes et les paragraphes courts

### Le Z-pattern

Le Z-pattern est courant sur les pages simples ou marketing :

- landing pages
- pages d'accueil simples
- pages de présentation

Le regard suit un trajet proche de ceci :

```text
LOGO              MENU

      IMAGE

TEXTE           BOUTON
```

Le regard passe souvent :

1. en haut à gauche
2. en haut à droite
3. en diagonale vers le centre
4. vers le bas à droite

C'est pour cela qu'on place souvent :

- le logo en haut à gauche
- la navigation en haut à droite
- un bouton principal vers le bas à droite

## Les principes de Gestalt appliqués au webdesign

La psychologie de Gestalt explique comment notre cerveau organise spontanément les formes visuelles. Ces principes sont très utiles en UX.

### 1. Proximité

Les éléments proches sont perçus comme appartenant au même groupe.

```text
Titre
Description
Bouton
```

Si l'on augmente trop les espaces, le lien entre ces éléments devient moins évident.

### 2. Similarité

Les éléments similaires sont perçus comme appartenant à une même catégorie.

Exemple :

```text
Carte
Carte
Carte
```

L'utilisateur comprend tout de suite qu'il s'agit d'éléments comparables.

### 3. Continuité

L'oeil suit naturellement une ligne ou une direction.

```text
Étape 1 -> Étape 2 -> Étape 3
```

Ce principe est utile pour :

- les formulaires
- les tunnels d'inscription
- les étapes d'achat

### 4. Fermeture

Le cerveau complète naturellement une forme incomplète.

```text
(   )
```

On reconnaît un cercle même s'il n'est pas entièrement dessiné.

### 5. Figure / fond

Le cerveau distingue spontanément un élément principal de son arrière-plan.

Exemple : un bouton sur un fond neutre ressort immédiatement comme un élément interactif.

## Erreurs de design fréquentes chez les développeurs

Les développeurs se concentrent souvent sur :

- la technique
- la logique
- les fonctionnalités

Mais pas toujours sur l'expérience utilisateur. Certaines erreurs reviennent souvent.

### Menus trop complexes

Un menu avec trop d'entrées crée une surcharge cognitive.

Exemple :

```text
Accueil
Produits
Produits premium
Produits standard
Offres
Offres spéciales
Services
Services entreprises
Services particuliers
Blog
Actualités
Contact
À propos
```

L'utilisateur ne sait plus où cliquer.

Bonne pratique : limiter le menu principal à 5 ou 6 entrées maximum.

### CTA invisibles

Un bouton important doit être immédiatement visible.

Erreurs fréquentes :

- bouton de la même couleur que le fond
- bouton perdu dans le texte
- bouton trop petit

Un CTA efficace doit :

- contraster
- être visible
- être compréhensible

### Trop d'informations

Les développeurs veulent souvent tout afficher, mais l'utilisateur cherche surtout à comprendre rapidement l'essentiel.

Une page efficace doit répondre rapidement à la question : pourquoi suis-je ici ?

### Mauvaise hiérarchie des titres

Si plusieurs titres ont la même importance visuelle, la structure devient confuse.

Exemple d'organisation claire :

```text
Titre principal
Sous-titre
Section
Sous-section
```

### Formulaires trop longs

Un formulaire trop long augmente le risque d'abandon.

Bonne pratique :

- réduire les champs
- regrouper les informations
- afficher les étapes si nécessaire

### Manque d'espace

Remplir tout l'écran rend l'interface plus stressante et plus difficile à lire.

L'espace vide n'est pas une perte de place. Il aide à respirer, structurer et comprendre.

## Résumé

Un bon design guide l'utilisateur grâce à :

- l'équilibre
- l'alignement
- la proximité
- la répétition
- le contraste
- la hiérarchie visuelle
- les parcours de lecture naturels
- les principes de perception visuelle

L'objectif n'est pas de montrer tout ce que le site peut faire.

L'objectif est de permettre à l'utilisateur d'atteindre rapidement ce qu'il veut faire.
