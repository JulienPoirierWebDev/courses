# Mode opératoire

## Remplir le tableau de synthèse – Épreuve E5 (BTS SIO)

## 1. À quoi sert ce document

Le **tableau de synthèse E5** sert à montrer au jury :

* les **réalisations professionnelles** que vous avez effectuées
* les **compétences du référentiel BTS SIO** que vous avez utilisées
* les **preuves disponibles dans votre portfolio**

C’est **le document que le jury lit pour choisir les questions à l’oral**.

Donc :

➡️ s’il est mal rempli, le jury ne voit pas votre travail
➡️ s’il est bien rempli, il valorise votre portfolio

Il doit permettre au jury de comprendre rapidement :

* ce que vous avez fait
* quand
* quelles compétences ont été mobilisées
* quelles preuves existent

---

# 2. Les informations à remplir en haut du document

### NOM et prénom

Nom complet du candidat.

### N° candidat

Numéro fourni par le centre d’examen.

### Centre de formation

Nom de l’établissement.

Exemple :

```
IPSSI Paris
```

### Option

Cocher :

* SLAM → développement

### Adresse URL du portfolio

Lien vers votre portfolio en ligne.

Exemples :

```
https://portfolio-jdupont.fr
https://github.com/jdupont/bts-sio-portfolio
```

Conseil important :

Le jury doit pouvoir **ouvrir ce lien facilement**.

---

# 3. Comprendre la structure du tableau

Chaque ligne correspond à **UNE réalisation professionnelle**.

Une réalisation peut être :

* un projet
* un TP important
* une mission en stage
* une contribution à un projet
* une mise en production
* un développement
* une installation technique

Il y a **trois grandes sections** :

### 1️⃣ Réalisations en formation

Projets faits en cours.

Exemples :

* API REST
* application web
* projet Symfony
* projet React
* mise en place Docker
* configuration serveur

---

### 2️⃣ Réalisations en stage (1ère année)

Missions réalisées en entreprise.

Exemples :

* correction de bugs
* création de script
* maintenance serveur
* ajout de fonctionnalité
* déploiement d’application

---

### 3️⃣ Réalisations en stage (2ème année)

Même principe.

Ces missions sont souvent **les plus valorisées**.

---

# 4. Comment remplir la colonne

## "Réalisations professionnelles"

Cette colonne doit contenir :

1️⃣ **le titre de la réalisation**
2️⃣ **une description courte**
3️⃣ **les preuves disponibles**

Structure recommandée :

```
Titre de la réalisation

Description rapide

Documents associés :
- code source
- documentation
- capture écran
- schéma
```

---

### Exemple SLAM

```
Entreprise XX - Développement d'une API REST pour la gestion de tournois

Création d'une API REST en Node.js avec Express et Sequelize
permettant la gestion des clubs et tournois sportifs.

Documents associés :
- dépôt GitHub
- documentation API
- schéma UML
- captures Postman
```


### Lien avec le portfolio
Vous ne pouvez que cocher des cases sur le tableau de synthèse. Donc il faut impérativement que l'on retrouve ces projets sur votre portfolio. Je vous conseille de suivre la structure suivante : 

Voici une structure type : 
- Contexte d’entreprise et de la mission
- Mini cahier des charges (définition des besoins) 
- Développement de la solution
- Listes des outils
- Compétences développées.

# 5. La colonne

## Période

Format obligatoire :

```
JJ/MM/AA au JJ/MM/AA
```

Exemple :

```
05/01/25 au 20/02/25
```

Conseil :

La période doit correspondre :

* au stage
* ou à la période du projet

# 10. Les erreurs fréquentes

### ❌ erreur 1

mettre des descriptions trop longues

➡️ rester synthétique

---

### ❌ erreur 2

ne pas mettre les preuves

➡️ le jury doit voir où vérifier

---

### ❌ erreur 3

cocher toutes les compétences

➡️ rester honnête

---

### ❌ erreur 4

mettre des TP insignifiants

➡️ choisir des projets **significatifs**

---

# 11. Bonnes pratiques

# 12. Vérification finale avant l'examen

Vérifier que :

✔ toutes les lignes ont une période
✔ toutes les réalisations ont des preuves
✔ les compétences sont cohérentes
✔ le portfolio contient les documents cités

* votre méthode
